EECS 110 Dev-C++ Graphic Library Notes

This page describes how to create projects in Dev-C++ for Windows that use the EECS 110 Graphics Library for simple graphics. The instructions assume you have already installed the Dev-C++ library. If you haven't, first follow these instructions.

Creating a EECS 110 Graphics Project

A graphics project in Windows is one that opens a special graphics window in which drawing and mouse actions can happen. Microsoft provides C/C++ functions for doing this but they are quite complicated to use. The EECS 110 introGlutLib library makes it possible to do simple graphics with less complexity.

Start Dev-C++. Click on its File menu, then on New, then on Project ....

Click on the image to enlarge it.

The New Project dialog box will appear. Click on the tab at the top labelled Introduction. This will show project templates designed for introductory programming. You should see one labelled CS110-Graphics. If you do not, cancel, and re-do the steps for installing the EECS 110 files.

Click on the image to enlarge it.

Select the CS110-Graphics option and enter a name for your project. Here I used PA3. Click Ok to create the project. As before, Dev-C++ will ask you where to save the project. As always, create a new folder in your EECS 110 code folder, e.g., PA3, open it, and put your project there.

Click on the image to enlarge it.

When the project opens, it will contain several (unsaved) files. Three of these contain the graphics library code. You can look at them, but don't edit them. The fourth, main.c, is where you should put your code for drawing, handling mouse clicks, and handling key clicks.

Click on the image to enlarge it.

Test your new project immediately! Click the icon to Compile and Run the project, press the F9 function key, or select "Compile and Run" from the Execute menu, whichever you prefer.

Dev-C++ will ask you where it should save those files. Just hit Enter each time to save them in the project folder you just created.

Dev-C++ will then try to compile and run the new project. If everything is correctly installed, compilation should run without errors, and the application should start and show a green rectangle.

Simply click the mouse or press a key to exit the program. You're now ready to add your own code. If anything fails, re-check the steps above, read the error message, ask your partner to help, and if all else fails, post the error messages to the newsgroup.

If the new project compiled and ran correctly, then create your own code by replacing the code in main.c. Only change the functions you need to, e.g., myDisplay() for drawing things, myMouse() if you're doing anything with mouse clicks, and so on. Remember to always leave at least one way to exit your program!
