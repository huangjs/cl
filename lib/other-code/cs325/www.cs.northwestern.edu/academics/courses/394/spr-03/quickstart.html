Software Project Management

Goal

This course is about becoming a professional software developer, not just a programmer. The course focuses on 3 topics most critical to project success or failure and least familiar to computer science and engineering students:

  • Serious testing of code
  • Requirements gathering and analysis
  • Project planning

We'll spend approximately 1/3 of the course on each topic, and each topic will lead to a deliverable. Programming is a major part of testing, but not of the other 2 topics.

Prerequisites

Significant project development experience, such as can be found in a multi-quarter CS 399 development project, CS 343 (Operating systems) or similar courses. Industry experience is not required but very helpful.

In addition, you must have ready access to the university network and to email.

Textbooks

There are two textbooks for this course:

  • William Wake's Extreme Programming Explored: This will drive the unit on testing, and also introduce you to agile software development.
  • Pankaj Jalote's Software Project Management in Practice: This will drive the unit on project planning and management.

These are both short very practical books. They should be available at the university book store, as well as at Amazon and other online sellers.

The requirements topic will be use notes and online resources.

Grade Items

There are three core class activities:

  • Testing and code review: You'll produce an in-depth test suite for a body of code. You also participate in structured walkthroughs on such code.
  • Requirements analysis: You'll create a preliminary requirements analysis document for a medium size project, using standard UML notations.
  • Project plan: You'll create a risk-driven iterated project development plan for the project.

You will be graded on:

  • Independent initiative in tracking down materials in each of the theme areas,
  • Quality of materials submitted (programs and reports), and
  • Class participation, including
    • brief presentations of materials you've found in texts, journals, and the net
    • well-reasoned commentary and questions on such presentations

There are no exams.
