Common mistakes:

  • Useless tests, e.g., an assertion that getName() is not "John" following an assertion that getName() is "Mary" -- the only way that could fail is if "John" = "Mary" which is a language bug.
  • Code without tests, e.g., interface code for which no tests were written
  • Misused assertion forms, e.g.,
    • assertEquals(true, ...) instead of assert(...)
    • assertEquals(false, ...) instead of assert(!...)
    • assertEquals(actual, expected) instead of assertEquals(expected, actual) -- this will cause misleading messages on failure
    • assert(x.equals(y)) instead of assertEquals(y, x) -- often people did this because they didn't realize that assertEquals() needs 3 arguments for floating point numbers
